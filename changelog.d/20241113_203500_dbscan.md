### added
- Added DBSCAN clustering support to the core `cluster_microstates` API, MSM discretizer, and both Streamlit and CLI workflows so density-based microstates can be requested wherever KMeans was previously the only option. Users can now provide DBSCAN-specific kwargs (e.g. `eps`, `min_samples`) through configuration, the web UI, or CLI arguments.
- Introduced `example_programs/conformations_dbscan_full_dataset.py` to reproduce the webapp's conformations workflow (20 runs / 33 shards) with the DBSCAN preset so regressions like the negative-index failure can be validated in isolation. The script saves artefacts under `example_programs/programs_outputs/conformations/` and now emits the same deliverables as the Streamlit tab: PCCA overlays, TPT plots, and detailed conformation summaries.
- Added a `--allow-disconnected-msm` flag to the headless analysis CLI so scripted runs can explicitly bypass the fully connected guardrail (matching the `BuildConfig` switch exposed to the backend and UI).

### changed
- Updated conformations analysis, web app forms, and CLI tooling to treat `kmeans_kwargs` as general cluster kwargs, including smarter handling of `n_init` and seed parameters when DBSCAN is selected.
- Streamlit conformations and MSM/FES tabs now expose algorithm-specific controls (including dedicated eps/min-samples inputs for DBSCAN) instead of a generic JSON field, and the CLI surfaces matching DBSCAN flags for parity.
- Progress feedback in replica-exchange loops now relies on `tqdm`, giving the heating, temperature equilibration, and production stages richer ETA/readability without maintaining a bespoke progress bar.
- File/name slugification now delegates to `python-slugify`, ensuring consistent, Unicode-aware filename sanitization without our bespoke ASCII regex.
- Standardized every MSM clustering call on scikit-learn's `KMeans`/`MiniBatchKMeans`, removed the old deeptime-only kwargs, and refreshed the README/web controls to document the supported algorithms and parameters consistently.
- JSON serialization helpers now lean on Pydantic's adapters instead of bespoke recursion, giving consistent handling for numpy objects, Paths, and guardrails around NaN/Inf while keeping downstream callers unchanged.
- Logging time utilities now lean on `humanize` for duration rendering and `contexttimer` for the stage timer, eliminating the custom formatting/timing code paths while keeping the same banner UX.
- `pmarlo.api.normalize_training_metrics` now strictly validates the best score/epoch/tau fields instead of guessing them, so consumers get immediate feedback when histories are incomplete.

### fixed
- Resolved a regression where the Streamlit conformations workflow crashed before running DBSCAN because the selected cluster method was referenced prior to being assigned, preventing the new density-based option from running end-to-end.
- DBSCAN clustering now auto-tunes its `eps` radius from the reduced feature data (while respecting user overrides) and the conformations backend raises a descriptive validation error when the resulting microstates are insufficient for the requested metastable states, avoiding opaque one-state MSM failures in the web app.
- Restored the `_compute_analysis_diag_mass` export, implemented the missing `_covariance` helper, and made diagnostics raise `InsufficientSamplesError` for undersized splits so analysis guardrails/tests stop failing on synthetic data and canonical shard IDs.
- Fixed the conformations flux ranking to average per-edge incoming/outgoing flow and to use the same metric in both the finder and headless TPT analysis so bottleneck detection no longer double-counts degrees in tests or the UI.
- Made `pmarlo.features` truly lazy about importing mdtraj (with a test hook to simulate missing installs), ported the DeepTICA trainer facade to accept dummy models, preserve the `tau_steps` input, and expose legacy `step` metrics, so the lightweight feature tests run without the heavy ML stack.
- Simplified JSON sanitisation and removed the recursive `TypeAdapter` usage that triggered pydantic recursion errors, keeping structured summaries intact without optional dependencies.
- Added a built-in slugifier fallback and restored `_initialise_torch_force` as a wrapper over the legacy initializer so replica-exchange validation tests can monkeypatch the OpenMM torch hooks without installing `python-slugify`/`openmmtorch`.
- Defaulted the DeepTICA curriculum trainer to use CPU devices and propagated the legacy `tau_steps`/`step` APIs through the facade so curriculum tests no longer choke on the removed "auto" device mode.
- Allowed the settings loader to honor the `HOME` override when expanding ``~`` in `feature_spec_path`, fixing replica-exchange configs on Windows and in tests.
- Canonical shard IDs now omit the `_default` suffix when no run identifier is provided, so synthetic shards created in tests continue matching the validation expectations.
- The diagnostics directory resolver now prefers `example_programs/programs_outputs/diagnostics` before falling back to `example_programs/diagnostics`, matching the Streamlit example workflow layout.
- Replaced the scikit-learn-dependent PCA/DBSCAN/canonical-correlation pieces with numpy implementations and fallbacks so MSM clustering and analysis work in environments that lack the heavy ML stack.
- Strengthened `tests/test_visualization_diagnostics.py` to fully exercise the sampling, FES, and shard histogram helpers so regressions in the plotting diagnostics surface immediately instead of slipping past the suite.
- Reworked the conformations backend and REMD initialization tests to assert user-visible artifacts (plots, representative PDBs, and automatic setup state) instead of tracking helper calls, making the suite resilient to refactors while still guarding behavior.
- DeepTICA training finalisation now records definitive best validation score/epoch/tau values for the mlcolvar and deeptime backends, eliminating the fragile heuristics that previously tried to reconstruct them after the fact.
